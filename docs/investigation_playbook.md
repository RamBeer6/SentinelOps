# Investigation Playbook

This playbook describes a practical investigation workflow for the alert types generated by SentinelOps.

## Brute Force Alert

1. Identify the source IP.
2. Count failed authentication attempts.
3. Review targeted users.
4. Check whether the source IP is trusted.
5. Search for successful logins from the same source after the failures.
6. Recommend blocking, rate limiting, MFA enforcement, or password reset actions.

## Suspicious IP Alert

1. Identify the source IP and related activity.
2. Determine whether the IP is internal, external, or explicitly trusted.
3. Review accessed endpoints and authentication outcomes.
4. Check whether the same source appears in other alerts.
5. Recommend watchlisting, blocking, or endpoint access restrictions.

## Privilege Access Alert

1. Identify the accessed path.
2. Identify the user and host involved.
3. Review process activity around the same timestamp.
4. Assess asset criticality.
5. Confirm whether the access was expected administrative activity.
6. Recommend permission review, host telemetry collection, or containment if suspicious.

## Abnormal Behavior Alert

1. Identify the process, host, and CPU value.
2. Review the parent process and command line where available.
3. Check whether the host also has sensitive file access or authentication alerts.
4. Determine whether the behavior matches expected workload.
5. Recommend process termination, host investigation, or workload tuning.

## Correlated Attack Alert

1. Review all underlying detections.
2. Confirm shared source IP or host.
3. Prioritize any successful authentication events after failed attempts.
4. Assess the affected assets and users.
5. Escalate to incident response if the activity is confirmed suspicious.
6. Recommend blocking, credential review, endpoint isolation, and wider log search.
